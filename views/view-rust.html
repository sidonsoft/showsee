Rust

Systems Language Memory Safe No GC

What is Rust?

A systems programming language that gives you control over memory without the danger of C/C++. No garbage collector, no runtime overhead.

Created by Mozilla in 2010 to solve a specific problem: writing browser components that are fast AND safe.

The key innovation: the compiler enforces memory safety. If your code compiles, it's free from whole classes of bugs (buffer overflows, null pointers, data races).

How It Works: The Ownership System

Rust uses three rules enforced at compile time:

1. Ownership

Every piece of data has one "owner". When the owner goes out of scope, the data is freed. No garbage collector needed.

2. Borrowing

You can lend data to functions without giving away ownership. The compiler tracks references and enforces rules.

3. Lifetimes

Ensures references don't outlive the data they point to. Prevents dangling pointers.

The "borrow checker" is what makes Rust unique. It rejects code that could cause memory errors — at compile time, not runtime.

Code Example

// Ownership: s is moved, not copied let s1 = String::from("hello"); let s2 = s1; // s1 is moved to s2 // println!("{}"); // ERROR: s1 is invalid // Borrowing: lend without moving let s3 = String::from("world"); let len = calculate_length(&s3); // & means borrow println!("{} has {} chars", s3, len); // s3 still valid

Who Uses It?

Mozilla Amazon (Firecracker) Google (ChromeOS, Android) Microsoft (Windows) Discord Dropbox Cloudflare Tesla Linux kernel

What It's Used For

  • Operating Systems — Linux kernel drivers, Redox OS
  • Web Browsers — Firefox components, Servo engine
  • Command Line Tools — ripgrep, fd, bat, exa (10-100x faster)
  • WebAssembly — run Rust in browsers at near-native speed
  • Networking — high-performance servers, proxies
  • Embedded — microcontrollers, no_std environments
  • Games — Bevy engine, game tooling
  • Blockchain — Solana, Polkadot, Ethereum clients
  • AI/ML — inference engines, ML frameworks

Should You Learn Rust?

Yes if you:

  • Want C/C++ performance with safety guarantees
  • Build systems where bugs are expensive
  • Enjoy solving puzzles (the borrow checker)
  • Care about long-term code quality

Maybe not if:

  • You need fast iteration / prototypes
  • Your team can't handle the learning curve
  • Python/JS is good enough for your use case
